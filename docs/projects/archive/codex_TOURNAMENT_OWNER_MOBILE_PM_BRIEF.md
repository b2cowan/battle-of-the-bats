# Tournament Owner/Admin Mobile PM Brief

Started: 2026-05-24

## Proposed functionality

Improve the mobile admin experience for organization owners/admins running the base Tournament subscription. The first implementation wave focuses on shared mobile foundations: bottom navigation overlap, touch target sizing, venue management, admin-entry recovery, safer activation, and mobile-friendly locked states.

## Why it matters

Tournament owners are likely to use phones during setup conversations and tournament-day operations. Base Tournament should feel complete and credible on mobile. If key actions are covered by navigation, controls are too small, or Plus locks look like broken features, owners may lose confidence before they reach the value of the product.

## Customer impact

Base Tournament owners will be able to:

- Get into tournament admin without being stuck on a loading handoff.
- Tap controls comfortably on a phone.
- Use registration bulk actions without the bottom nav blocking them.
- Manage venues from mobile without a desktop table squeeze.
- Avoid accidental tournament activation from the mobile More menu.
- Understand locked Plus tools through visible explanations, not hover-only hints.

Tournament Plus remains positioned as the operations upgrade for exports, automation, custom fields/files, branding, cloning, targeted communication, and post-event reporting.

## Priority

High. Tournament is the live-selling product path, and mobile admin quality affects setup completion, tournament-day confidence, and conversion from base Tournament to Tournament Plus.

## Success criteria

- P0 shared mobile issues are implemented before broad page-by-page redesign.
- Base Tournament workflows remain primary on mobile.
- Plus gates are compact, contextual, and understandable before a user taps.
- Browser verification at 390x844 shows no bottom-nav overlap, no obvious horizontal overflow, and comfortable touch targets on the highest-use admin pages.
