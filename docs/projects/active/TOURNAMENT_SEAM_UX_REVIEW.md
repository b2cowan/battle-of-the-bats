# Tournament Seam UX Review — Coaches, Admins, Scorekeepers ↔ the Public Tournament

**Date:** 2026-07-21 · **Method:** two multi-agent adversarial reviews (135 subagents total: surface mapping → click-path scenario audits → dedup → 2-lens code-grounded verification per finding → completeness critic). Every finding below survived an independent verification pass that read the cited code and actively tried to refute it; severities shown are **post-adjustment** (several were lowered when a verifier found an existing mitigating affordance, a few raised).

**Scope:** the "1 app" seam — how a tournament coach (free + Premium), an org admin/staff member, and a volunteer scorekeeper move between the **public tournament surface** (`/{orgSlug}/{tournamentSlug}/*`) and their **working shell** (coach workspace `/coaches/team/*`, Premium portal `/{orgSlug}/coaches/*`, admin `/{orgSlug}/admin/*`, scorekeeper `/{orgSlug}/scorekeeper`), plus the global consumer chrome (Home/Scores/Chat/Account) and chat round-trips. Mobile and desktop.

**Companion detail:** full per-finding claims, code evidence, and verifier commentary are in the appendices at the bottom (generated from the verified workflow output). Raw workflow transcripts: session scratchpad `coach-review-digest.json` / `operator-review-digest.json`.

---

## Executive summary

The "1 app" foundation is genuinely in place: the persistent Home/Scores/Chat/Account bar survives on public tournament pages, the account-sheet **hat chip** on tournament pages is a real working bridge for every role that resolves a hat, and no persona ever needs to remember a URL. Getting *into* a working shell is generally 1–2 taps.

The seams fail in four repeated ways:

1. **Chat is a hub with no spokes back** — the single worst "1 app" failure, affecting every persona (fans included).
2. **Doors land in the wrong room** — bridges exist but drop context (land on dashboards/roots, not the page-matched screen) and are asymmetric about tab handling.
3. **The words on coach doors are wrong** — "Schedule" and "Fees/Dues" in both coach shells point at self-entered data, not the tournament's; the paid (Premium) tier is the *worse* half of the seam.
4. **Game-day resilience gaps** — session expiry silently eats a scorekeeper's entered score; notification deep-links land on the marketing homepage; no offline story on any authed shell.

## Theme 1 — Chat: easy in, no way back (all personas) — HIGH

- The persistent Chat tab always goes to the bare global inbox; all tournament context is dropped on entry.
- An open room has **zero** links back to its tournament: the room's event-name header is plain text; "Back" returns to the flat inbox. Confirmed for coaches, admins, staff, fans; mobile + desktop. The API already delivers the event identity to the client — it is simply never rendered as a link.
- Org admins/owners are auto-enrolled as room moderators server-side, and the consumer room view even receives the moderator flag — but the consumer chat view never wires the moderation actions, so an admin in the global chat has no pin/delete/mute; full powers exist only on the admin chat page, which has no cross-link from the room.
- **Chat push notifications carry no deep link** — tapping a `chat_mention`/`chat_message` push opens `/` (the anonymous marketing homepage for signed-in users, since `/` only auth-redirects with `?source=pwa`). The one notification meant to pull people into chat mid-tournament strands them at the front door.
- Chat return support is asymmetric: inside a coach shell, persistent shell chrome keeps a 2-tap route back; the public/global chat surface unmounts the tournament bar entirely.
- Polish: the inbox never auto-opens a lone room (coach portal chat does).

## Theme 2 — Doors land in the wrong room (operators, coaches)

- **Hat chip is the bridge and it works** (2 taps from any tournament page to Open admin / Coach view / Scorekeeper) — but every hat href is a static, context-free root: admin → dashboard (never the page-matched Results/Schedule), coach → workspace overview/portal root, scorekeeper → today's unfiltered list (no game/date params; the page never reads searchParams). No return params anywhere; browser-back happens to work public→admin but nothing surfaces it.
- **Tab-handling asymmetry is backwards:** public→admin hat links navigate same-tab (replacing the public page the admin was sharing/watching), while admin→public "View Site" opens a new tab. On an installed PWA the target=_blank breakout dumps the operator into the system browser with no way back (ratified game-day mode).
- "View Site" always lands on the tournament **Home** tab, never the schedule/results page the admin was verifying (+1 tap every trip).
- Draft-tournament fork: sidebar/bottom-nav correctly swap preview↔live by status, but the org tournament-list preview icon **always** opens the admin preview regardless of live status; the preview shell itself has deliberately zero return-to-admin affordance, and a permission-denied/stale preview link 404s bare with no way back.
- Results page on mobile hides finalized games by default with no hint (desktop shows a "Completed N" chip) — the exact "fix a wrong finalized score" entry is invisible on first arrival.
- Home's workspace card ignores tournament context (localStorage/active-flag tournament, not the one just viewed) — mitigated by the chip on the tournament page itself.

## Theme 3 — Coach doors are mislabeled; Premium is the orphan

- **"Schedule" is a dead end in both coach shells** (Premium: fixed #2 bottom tab): self-entered calendar only, no feed from and no link to the real tournament schedule, which lives under "Tournaments" (no time-word on the door). High for Premium, Medium for free.
- **Money words point at the wrong ledgers.** Free overview "Fees" tile = coach-authored parent-billing ledger → shows **"$0.00 · 0 unpaid"** while the real tournament fee is past due (false all-clear on the exact screen "check my dues" lands on). Premium "Money → Player Dues" = unrelated internal ledger, no cross-link. The real fee status (amount, due date, organizer contact) renders on the tournament record — 2–3 taps behind doors with no money word. No pay button anywhere (verified as a deliberate read-only phase decision; Low).
- **Premium gaps:** overview hero CTA routes a tournament-bound coach into their empty personal calendar ("Tournament coming up" with no opponent/time/field); the Premium tournament record has **no link back to the public tournament site** mid-tournament (free tier has one); public-page coach detection misses Premium-only coaches (org-added / claim-token — no chip, no Coaches Portal link; today's claim-token path is platform-admin-seeded only, so the blast radius is currently small but grows with Founding Season self-serve); Premium portal has no consumer nav at all; the aggregate "fees due" flag is an unitemized lump sum; claim-token teams get an empty Tournaments tab (history lookup misses them though Chat resolves).
- Free-tier variants: overview tile silently drops the live tournament-game cue once any self-entered event exists; "Fees" can lose the scarce 4th mobile tab slot to a fixed priority list; "Fan view" backlink exists on only 2 of ~9 workspace pages and always lands on tournament Home.

## Theme 4 — Game-day resilience & trust

- **Scorekeeper session expiry is the worst single finding (High):** on a 401 mid-save the recovery notice (with the only sign-in link) renders *behind* the score sheet's opaque backdrop; Save re-fails silently and repeatably; closing the sheet to find the notice **discards the entered score**; re-auth lands on the full list, not the game. The check-in shell has the same class of problem (generic "Action failed.", no re-auth affordance).
- No offline fallback on any authed shell (coach/admin/scorekeeper are service-worker no-intercept routes): dead connection at a field = browser error page, not even a branded offline shell. (Public tournament pages do get the offline fallback.)
- Notification deep-links: chat pushes → `/` (above); `score_submitted` → the admin Schedule page, which has no score UI (review lives on Results) and no gameId; `score_disputed` is one of 5 declared-but-dead event types (no call sites — the "dispute" flow doesn't exist).
- Scorekeeper shell scoping: "assigned game" is a fiction — the shell shows the whole org/tournament day slate and "Up next" badges the earliest game anywhere, not the volunteer's; mitigated by inline venue/field per card + team search (which also answers the "what field is the 2pm game on" question without leaving).

## Theme 5 — Multi-hat identity fragmentation

- Home ranks the same account "Admin first" while the tournament chip badges it "· Coach" first — two first-party surfaces disagree on primary identity the same day.
- Admin shell has **no Coach-view door for Basic coaches** (the only Coaches-Portal link is Rep-Teams/paid-gated) — the small-club "I run the tournament and coach in it" case must exit to the public page and use the chip.
- The org Coaches Portal has no path back to org admin; mobile admin has no path to global Chat/Account/Home at all for single-workspace accounts (desktop link is ≥2-workspace-gated).
- Public schedule → admin chat has exactly one path (the chip) and it's hat-gated: unassigned staff and platform admins see no door on the page.
- Site root `/` is session-blind except for the PWA start-url marker (marketing page with generic "Sign In" for signed-in users).
- Refuted for balance: "Account tab is a dead end for operator hats" — the Account tab does carry working doors; and admin chat unread IS visible inside the tournament-ops subtree (non-issue, confirmed).

## Cross-cutting security/privacy flag (not UX, must fix)

- **Assistant-coach money-capability bypass:** an assistant explicitly denied money visibility still sees the team's real tournament fee owed/paid/past-due via the Premium tournament record page (no capability check on that route; nav item gated only on history existing).
- Also worth noting: registration never creates a follow (the public schedule can't highlight "your team" until a manual follow), and the "your team" widgets omit the **field** — the most venue-dependent fact — while showing rank.

## Recommended fix order (smallest effective change first)

**P1 — broken game-day loops**
1. Scorekeeper 401 recovery: surface the notice above the sheet, keep the entered score, re-auth back into the same game. Same pattern for check-in's failure banner.
2. Chat return path: render the room's event name as a link to the tournament (data already client-side); make the tournament-page Chat tab carry tournament context; add an "Open admin chat" affordance for moderators (or wire the existing moderation flag in the consumer view).
3. Chat push deep links: send the room URL in the push payload (both events); repoint `score_submitted` at Results with the game focused; delete or build the 4 other dead notification types.
4. Mobile Results: show the desktop's "Completed N" chip (or include finalized games with a status pill).

**P2 — re-point existing doors (label + landing fixes, no new surfaces)**
5. Coach "Schedule" tabs: feed or link the real tournament schedule (the tournament-record schedule component already exists in both shells).
6. Money doors: point the free overview money tile at the tournament fee (or show both rows); add a money-worded nav door to the tournament record; cross-link the two ledgers in both tiers; itemize Premium's lump sum.
7. Premium parity: always-on backlink to the public tournament from the tournament record; detect Premium-only coaches in the public-page hat resolver; re-point the overview hero CTA at the tournament (mirror the free tier's next-game tile).
8. Context-preserving doors: hat links deep-link to the page-matched screen with a return param; "View Site" preserves the current page; flip the tab asymmetry (public→admin new-tab or return-aware; suppress target=_blank inside installed PWA); status-aware preview icon in the org tournament list; a way out of the preview shell + a branded preview 404.

**P3 — cohesion & polish**
9. Auto-follow the coach's team at registration; add field/venue to the your-team widgets.
10. Multi-hat coherence: consistent primary-identity rule between Home and the chip; a Coach-view door in the admin shell for Basic coaches; mobile admin door to global Chat/Account; portal → admin backlink.
11. Scorekeeper scoping: pinnable field filter (or seeded assignment) so "Up next" is *their* next; deep-link the scorekeeper hat door with game/date.
12. Offline shells for authed routes; single-room chat auto-open; session-aware `/` (reuse the existing PWA redirect logic); capability-filter mobile admin primary tabs.

**Independent must-fix:** assistant-coach money bypass on the Premium tournament record (permission check, not UX).

### A1. Premium Overview CTA misroutes away from the tournament schedule
**Severity:** High · premium · both

**Claim (verified):** When a Premium coach has no self-entered event but has an upcoming tournament, the Overview hero's headline is content-free ('Tournament coming up', no opponent/time/field) and its primary lime CTA links to the coach's own empty self-entered calendar (`${base}/schedule`) instead of the tournament. The actual tournament schedule is NOT one tap away — the secondary link ('View tournaments') goes to a tournament list page (`${base}/tournaments`), and reaching the real per-game schedule requires a second tap into a specific registration's record page (`${base}/tournaments/[registrationId]`), which itself may show 'schedule not published yet' if the division hasn't published it.

**Evidence:** app/[orgSlug]/coaches/teams/[teamId]/page.tsx:944-964 (nowActions primary Link href=`${base}/schedule`, no opponent/time/field rendered)

**Verifier notes / fix direction:** Make the primary CTA 'View tournaments' (→ tournaments list) or, better, surface the actual next-game opponent/time/field in the headline sourced from the registration data, mirroring the free coach's C5 tile.


### A2. Premium coaches are invisible to the global Scores tab
**Severity:** Medium · premium · both

**Claim (verified):** 'coaches_premium' is not literally unreferenced in scores-feed.ts (it's used at line 440 to suppress duplicate org tiles) and the "invisible regardless of follow state" framing is false — a Premium coach who follows their team like any fan gets it in My Games via the standard follow path, and most real Premium coaches (upgraded from a Basic tournament registration) still surface via the surviving Basic identity that resolveCoachedTeams reads. The real, narrower gap: the 'coaches_premium' context itself carries no team/tournament linkage usable for Scores, so an org-added Premium coach with no underlying basic_coach_team and no personal follow of their team would not see that tournament auto-populate in My Games/My Events.

**Evidence:** lib/scores-feed.ts:134-155 (Basic-only comment + implementation); lib/user-contexts.ts:33-34,209,227,276 ('coaches_premium' kind exists but zero matches in scores-feed.ts)

**Verifier notes / fix direction:** CONFIRMED as a real code gap (coaches_premium is structurally absent from scores-feed.ts) but ADJUSTED down from High to Medium because the flow-lens picture is much softer than "invisible regardless of follow state": premium coaches already own a dedicated, arguably superior game-day home base (portal auto-redirect + 'Now'/next-game card, reachable in 1 tap from the Home tab's Workspaces card), and an explicit team-follow already produces a My-Games row today. Nothing strands the coach on game day. The suggested fix (mirror resolveCoachedTeams for rep_*/premium registrations) is directionally correct and the right long-term parity fix, matching the existing basic-coach resolver pattern — but it's not the cheapest fix for the sharpest pain. The sharper, smaller fix: (1) correct the Scores-tab empty-state copy (ScoresClient.tsx:177) to detect a coaches_premium context (already available via getUserAccessContexts, reused elsewhere) and point the coach at their Coaches Portal instead of the false 'no games' message, and (2) stop staffedOrgSlugs from suppressing a premium coach's own org tile in Scores since, unlike admin/official/basic-coach, no substitute surfaces there today. Build the full 1-tap 'My Games' resolver (the original suggestion) as the complete parity fix once the misleading-copy papercut is closed.


### A3. 'Schedule' nav tab is a dead end for real tournament games

**Evidence:** app/coaches/team/[basicTeamId]/schedule/page.tsx:22-38 (getBasicCoachTeamEvents only); grep of components/coaches/ScheduleEditor.tsx found zero Link/href to any org/tournament route

**Verifier notes / fix direction:** Confirmed as a real dead-end, but severity and fix must split by tier — the two portals are not symmetric, and a working "real schedule" mechanism already exists and just needs surfacing, not building.

PREMIUM (High confirmed): components/coaches/CoachesBottomNav.tsx:18-23 fixes exactly 4 mobile tabs — Overview, Schedule (#2), Chat, Roster. `/tournaments` is NOT one of them; it only appears inside the "More" sheet under a "Season" section header (line 38), and drops to a further-buried "Explore" sub-list when `hasTournamentHistory` is false (lines 82-88, lib/db.ts:4581). app/[orgSlug]/coaches/teams/[teamId]/schedule/page.tsx renders the self-entered RepTeamEvent calendar (tournament_game/practice/etc. are manually created via openAddForm, not fed from the registration) and contains zero links to /tournaments or any live-schedule affordance (grep confirmed no match). So a coach tapping the one persistent, always-on-screen "Schedule" tab on game day genuinely dead-ends, and the real fix (More → Season/Explore → Tournaments → CoachTournamentRecord) is 3+ taps deep and mislabeled relative to the coach's mental model. High is right here.

FREE TIER (adjust to Medium): app/coaches/layout.tsx → CoachPortalShell.tsx:37-49 splits nav into TIER1 = Overview/Tournaments/Chat (always visible) and TIER2 = Roster/Schedule/Fees/Announcements (opt-in, progressive disclosure — only appears once the coach activates it). The dead-end risk is much smaller here; a coach following "Tournaments" (the tab that matches how they got signed up) lands directly on the real feed.

REAL SCHEDULE ALREADY EXISTS — reuse it, don't rebuild: components/coaches/CoachTournamentRecord.tsx:473-474 renders a live `<CoachLiveSchedule>` fed by real `teamGames` (real opponents once `schedule_visibility === 'published'`), each game linking to `/${org.slug}/${tournament.slug}/schedule/${g.id}` (line 218). app/coaches/team/[basicTeamId]/tournaments/page.tsx:86-91 additionally shows an explicit "Fan view — public schedule & live scores" external link to the real public tournament page. Both free and Premium already route through this same CoachTournamentRecord component (app/[orgSlug]/coaches/teams/[teamId]/tournaments/[registrationId]/page.tsx:23-31), so the auto-updating feed is not missing from the codebase — it's just not reachable from the tab literally labeled "Schedule."


### A4. Free coach Overview 'Fees' tile is the wrong ledger, defaults to false all-clear
**Severity:** High · free · mobile

**Claim (verified):** The Overview page's TeamHQ 'Fees' tile (the first money figure a coach sees after 1 tap from Home) renders formatMoney(unpaidTotal) from getBasicCoachTeamFees — a self-entered, coach-authored ledger of money owed TO the coach BY parents/players. It has nothing to do with the tournament entry fee owed TO the organizer. Because fees are self-entered and most coaches never add any, this tile shows '$0.00 · 0 unpaid fees' even when the coach's real tournament fee is past due — a false all-clear on the very screen 'check my dues' lands on.

**Evidence:** components/coaches/TeamHQ.tsx:166-173 (StandaloneTeamHQ Fees tile, unpaidTotal prop); app/coaches/team/[basicTeamId]/page.tsx:126-127,178-179 (unpaidTotal computed from getBasicCoachTeamFees only); components/coaches/FeeEditor.tsx:251-254 ('not where you pay a tournament's entry fee'); real tournament fee lives only in TeamHQ.tsx:399-423 (tournament-variant fee strip) on a different page

**Verifier notes / fix direction:** Verified the full chain in code. `app/coaches/team/[basicTeamId]/page.tsx:118,126-127,173-183` always renders `TeamHQ variant="standalone"`, whose 'Fees' tile (`components/coaches/TeamHQ.tsx:166-173`) is `formatMoney(unpaidTotal)` from `getBasicCoachTeamFees` (`lib/basic-coach-fees.ts:4-19`), which its own header comment confirms is a coach-authored, PER-PLAYER charge ledger ('team dues owed by parents') — completely unrelated to the tournament entry fee. This tile sits in the primary Overview screen, 1 tap from the Coaches Portal home (`components/coaches/CoachPortalShell.tsx:37-41,176`), and 'Fees' is also reused as a Tier-2 nav label pointing at the same self-entered ledger (`CoachPortalShell.tsx:47`) — so the label collision is systemic, not a one-off typo. The REAL tournament fee (amountDue, past-due state, red/amber glance strip) only renders in the separate `TeamHQ variant="tournament"` hero (`TeamHQ.tsx:275-281,399-423`), reached via a different primary tab ('Tournaments' → tap a specific registration card, `app/coaches/tournaments/[teamId]/page.tsx` + `components/coaches/CoachTournamentRecord.tsx:441-453`) — an extra tap away, and nothing on the Overview page (including the 'Tournament history' list, which only shows a registration-status badge, `page.tsx:241-262`) hints that a real fee is owed. Since most coaches never populate the self-entered ledger, '$0.00 · 0 unpaid fees' is the default/common state — making the false all-clear the norm, not an edge case, on exactly the screen a phone-in-hand coach checks first. This is a genuine, high-stakes financial misread (risk: coach believes they're paid up on the tournament, isn't, and faces consequences from the organizer) directly undercutting the stated product goal of 'check my dues' being fast and trustworthy. High severity is correct.

Sharper fix: renaming the tile alone (e.g. 'My Fees'/'Team Fees') only fixes the label collision, not the missing-information problem — a coach could still read an accurately-labeled '$0 team fees' as reassuring while a real tournament fee sits past due one tab away. The better fix is to REUSE the already-built `CoachTournamentStatus` fee model (`lib/coach-status-model.ts:84`, already computes `status.fee.amountDue/state/isPaid`, already renders as a red/amber glance strip in `TeamHQ.tsx:399-423`) and surface that same compact strip on the Overview page whenever the team has an active accepted registration with `status.fee.hasSchedule && !status.fee.isPaid` — no new ledger/calc needed, just reuse of the existing pattern one page over — AND rename the self-entered tile (and its matching Tier-2 nav item) to disambiguate ('Team Fees' / 'Parent Fees') so the two concepts stop sharing the word 'Fees' anywhere in the free-coach nav.

### A7. No path back from an open tournament chat to that tournament's schedule
