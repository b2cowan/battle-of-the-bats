## Goal

Let a small house-league organizer run **one real season for free** — register players, build a single division of up to 8 teams, auto-generate a round-robin schedule, watch standings update, track fees by hand, and share one narrow public page — with the free size **enforced server-side**. League Starter is a **free-floor entitlement profile** layered on a normal free org (it lights up the house-league module + caps), **not** a new `OrgPlan` key. Ships as a **capped beta** behind a feature flag from a controlled `/start/league` create path. Also unblocks the user-journey audit's League/Club personas (the "big gap").

## PM Brief (condensed — full version in the paired brief)

**What it does:** Turns `/start/league` from a waitlist into a real, free, capped house-league season builder with one public schedule/standings/registration page.
**Why it matters:** Closes our market-floor gap (RAMP/TeamLinkt/Spond all run core for $0); makes paid League ("run a *bigger* league") an honest upsell; unblocks the audit's League/Club journeys.
**Who benefits:** Small house-league admins. No new roles. The free floor caps at 1 season / 1 division / 8 teams / manual fees / narrow public page / no full org site / no exports.
**Expected impact:** League-admin activation moves off ~zero; paid-League boundary becomes credible.
**Priority:** High (Launch Rail; parallelizable with the now-complete coach stream).
**Success criteria:** A small admin self-serves a usable season + shares a public page; caps hold against direct API calls; cap-hit/first-value events instrumented; no marketed CTA until caps proven.

## Locked decisions (ruled 2026-06-11)

- **Caps = 1 active season / 1 division / 8 teams**, teams-only (no separate registrant cap — per-division `capacity` + waitlist already bounds roster size).
- **Representation = a persistent `organizations.free_floor` column** (`'league_starter'`), decoupled from the timed `org_overrides` / `ENTITLEMENT_GRANTS_ENABLED` machinery. `plan_id` stays the paid ladder.
- **Beta = feature flag + unlisted `/start/league` create path** (self-serve by link, not marketed). Server-side flag gate on the create route too.
- **Narrow public page = in scope**, reusing the existing public league season pages; **no** branded full org site (`module_public_site` stays off).
- **Abuse controls = minimal now** (rate-limit creation + platform-admin visibility of free-floor creation & cap-hits); dormant-retention + duplicate-detection → Phase 8; **email-verify-before-publish stays in Phase 8's unified email-verification work.**
- **No payments** — manual paid/unpaid fee tracking only.

## Grounded current-state facts (the seams this plan builds on)

- **House-league is module-gated, not cap-gated.** Every `…/house-league/*` route gates on `hasCapability(role, caps, 'module_house_league')` **AND** `hasModuleEntitlement(org, 'module_house_league')` ([lib/module-entitlements.ts:14](lib/module-entitlements.ts#L14)). No season/division/team count limits exist → **caps are net-new**.
- **The capability axis is role-based, not entitlement-derived.** `hasCapability` returns `true` for `owner` unconditionally and house-league is role-defaulted for `league_admin` ([lib/roles.ts:78-87](lib/roles.ts#L78)). **So granting the module via the free-floor only needs to touch `hasModuleEntitlement` (the entitlement axis); no capability-side change.**
- **Module entitlement = plan modules ∪ `enabledAddons`** ([lib/module-entitlements.ts:14-20](lib/module-entitlements.ts#L14)). The free-floor profile must union in `module_house_league` here.
- **Cap pattern to mirror:** `getEffectiveTournamentLimit` ([lib/plan-config.ts:113](lib/plan-config.ts#L113)) + count-non-archived-and-return-403 ([app/api/admin/tournaments/route.ts:313-330](app/api/admin/tournaments/route.ts#L313), [clone route:284](app/api/admin/tournaments/[tournamentId]/clone/route.ts#L284)). Shape: `{ error: "Your plan allows …" }`, **status 403**.
- **Public league season pages already exist** and render off `module_house_league`: `/[orgSlug]/league` (hub) + `/[orgSlug]/league/[seasonSlug]/{,schedule,standings,register,status}`. The **branded org home** (`/[orgSlug]/page.tsx`) is the separate `module_public_site` differentiator; without it the org home shows a *simplified fallback* (tournaments + league + tryouts).
- **Exports are client-side download buttons** (CSV/XLSX/ICS) in the schedule + registrations admin pages — **no server export API**. "No exports" = gate those buttons.
- **Fee tracking is manual:** `league_seasons.registration_fee` is display-only; `league_registrations.registration_fee_paid` is a manual boolean. The full accounting **ledger** is the separate `module_accounting` (Club) module. Free-floor manual tracking = the paid/unpaid toggle, **not** the accounting suite.
- **The league onboarding wizard already exists** (season → divisions → registration → review) at [app/[orgSlug]/admin/onboarding/page.tsx](app/[orgSlug]/admin/onboarding/page.tsx) but auto-triggers only when `plan_id === 'league' || 'club'` (~L520) → **must move to an entitlement check**.
- **`/start/league` today** is a pure `EarlyAccessModalTrigger` waitlist; the mirror create pattern is `/start/tournament` → `AddOrgForm` → `POST /api/org/create` (creates a free `tournament`-plan org + founding comp). Platform-admin guards already on all `/start/*` routes + create APIs.
- **Next free migration = 125** (124 is the highest; Phase-5 gate 114–124 already in prod).

## Architectural Decisions

- **Decision:** League Starter = a persistent `organizations.free_floor` column, **not** a new `OrgPlan` key, **not** the timed `org_overrides` grant system. **Rationale:** matches the locked §12 model; keeps `plan_id`/`PLAN_CONFIG`/rank/checkout/pricing untouched; decouples a free-*forever* floor from the *timed*-grant machinery (which is OFF by default and semantically wrong for this).
- **Decision:** House-league is unlocked by unioning `module_house_league` into `hasModuleEntitlement` for free-floor orgs; **not** `module_public_site`. **Rationale:** the entitlement axis is the only gate that needs changing (capability axis is role-based and already passes for owner/league_admin); withholding `module_public_site` is exactly what keeps the full branded site a paid-League differentiator while the public *season* pages still work.
- **Decision:** Caps live in a new `lib/free-floor.ts` and are enforced by count-and-403 in the create APIs, mirroring `getEffectiveTournamentLimit`. **Rationale:** consistency with the proven tournament-slot pattern; UI hiding is insufficient (D6).
- **Decision:** Reuse the existing league onboarding wizard + public league pages; the only structural change is the wizard trigger (plan-check → entitlement-check). **Rationale:** minimal net-new surface; the season-builder and public pages already exist behind the module.
- **Decision:** Beta = feature flag + unlisted create path; paid-League CTA = express-interest (League is early-access). **Rationale:** prove caps before marketing (Phase 9 owns the flip); no checkout exists for early-access League.
- **Decision:** No migration touches `league_*` tables — only the `organizations.free_floor` column. **Rationale:** caps are computed/enforced in code from existing row counts; the league schema is unchanged.
