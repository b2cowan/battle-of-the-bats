# Help Center Platform Admin PM Brief

## Proposed Functionality

Upgrade Help into a searchable, task-based help center. Customer admins get clearer "how do I" navigation for the modules they can access. Platform employees get the same customer guide content plus a protected Platform Admin Operations section for internal support workflows.

Follow-up improvement: add "Getting Started By Role" paths so users can choose their job role first, then follow the key help recipes in the right order.

## Why It Matters

The old Help hub was organized mostly as module cards, which made it hard to answer practical questions like "how do I reset a password?" or "how do I temporarily give a customer access to a module?" This change makes Help useful for actual support work and reduces the need to remember where each admin tool lives.

## Customer Impact

Customers should find module setup, role, billing, and export guidance faster. The customer-facing guides now include practical task recipes for the common "how do I" jobs across Org Admin, Tournaments, House League, Registrations, Rep Teams, Coaches Portal, and Accounting. They still only see help for their role and enabled modules.

Role paths give owners/admins, tournament admins, league admins, registrars, rep program admins, coaches, and treasurers a guided first path into those recipes.

## Employee Impact

Platform employees can use one Help Center to support customers and follow internal SOPs for password resets, billing overrides, module overrides, audit investigation, retention, product packaging, and platform user access. From the Platform Admin sidebar, Help opens in a separate browser tab/window and the help pages omit the platform-admin side navbar so articles have a dedicated reading workspace.

Platform employees also get role paths for Platform Support, Billing/Product Admin, and customer-support reference workflows.

## Access Differences

- Customer help stays under the org admin area.
- Platform admin help stays under `/platform-admin/help`.
- Internal Platform Admin Operations documentation is visible only to platform-admin users.

## Priority

High. This improves support readiness before broader customer onboarding and gives employees a reliable place to find operational instructions.

## Success Criteria

- Help search returns relevant task and guide results.
- Platform employees can find password reset, billing override, module access, and bulk operation instructions from quick links.
- Customer-facing docs remain available without exposing employee-only SOPs to customers.
- Platform-admin Help opens in a separate reading workspace without the platform-admin side navbar.
- Help hub users can choose a role-based getting started path and land directly on relevant task recipes.
- The project is accepted as complete; remaining article and UI refinements should be driven by real support/customer usage patterns.
