# New Machine Setup — FieldLogicHQ Dev Environment

Runbook for standing up this repo's full dev environment on a fresh Windows
machine, ordered so **Claude Code is running as early as possible** and can
drive the remaining installs.

> **Key concept — the path-derived memory folder.** Claude Code stores this
> repo's chat history *and* project memory under
> `~/.claude/projects/<path-hash>/`, where `<path-hash>` is the repo's absolute
> path with the drive letter lowercased and every `:`, `\`, and space replaced
> by `-`. For `c:\Users\Robert Cowan\Documents\tournament-website` that is
> `c--Users-Robert-Cowan-Documents-tournament-website`. Use the **same Windows
> username and same repo location** on the new machine and history/memory map
> automatically. If the username differs, rename that one folder to match the
> new path (see Stage 1.5).
>
> **Microsoft-account caveat.** Signing into a new PC with the *same* Microsoft
> account does **not** reproduce the `C:\Users\<name>` folder — Windows derives
> it from the email (often the first ~5 characters), so a profile that was
> `Robert Cowan` (a local-account name, with a space) will likely come back as
> something like `rober`. MS accounts also frequently redirect `Documents` into
> OneDrive. So expect the repo's absolute path to differ, plan on the Stage 1.5
> rename as the normal case, and **do not clone into a OneDrive-synced folder**
> (syncing `node_modules`/`.next`/`.git` causes file-lock corruption).

---

## 0. Before leaving the old machine — copy the secrets

These hold live credentials or local-only state and cannot be regenerated by a
clone. Use a USB drive or secure transfer; avoid cloud sync for the secret
files where possible.

| From the old machine | What it is |
| --- | --- |
| `Documents\tournament-website\.env.local` | Dev secrets (Supabase, Stripe test, Resend, VAPID, UAT logins). **Gitignored — does not travel via git.** |
| `Documents\tournament-website\.env.production.local` | Prod Supabase service-role + live Stripe key. **Gitignored.** |
| `C:\Users\<you>\.aws\` (whole folder) | AWS CLI credentials + config — **may not exist.** On this setup the creds live in **User-level env vars** (`AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` / `AWS_DEFAULT_REGION=us-east-2`), not a file. If `.aws\` is absent there is nothing to copy — re-auth on the new machine (Stage 3). |
| `C:\Users\<you>\.claude\projects\c--Users-Robert-Cowan-Documents-tournament-website\` | Claude chat history + project memory for this repo |
| `C:\Users\<you>\.claude\settings.json`, `commands\`, `plans\` | Claude global settings, custom slash-commands, saved plans |

Do **not** copy `node_modules`, `.next`, or `.git` — those are rebuilt/cloned.

**Also: commit and push all in-flight work to `dev` first** (working-tree
changes and untracked files do not travel via a clone — only what is pushed to
`origin/dev` does).

## Stage 1 — Bootstrap (manual, ~20 min): get Claude running

The only part done entirely by hand. Goal: Claude Code live inside the repo.

1. **VS Code** — `winget install Microsoft.VisualStudioCode`
2. **Node.js LTS** — `winget install OpenJS.NodeJS.LTS` (target Node 24.x to
   match the known-good toolchain). Reopen the terminal so PATH refreshes.
3. **Git** — `winget install Git.Git`
4. **Clone to a non-OneDrive path.** A dedicated dev directory avoids OneDrive
   sync corruption:
   ```powershell
   mkdir C:\dev -Force; cd C:\dev
   git clone https://github.com/b2cowan/battle-of-the-bats.git tournament-website
   ```
   The first git auth opens a browser login (Git Credential Manager) — sign in
   as `b2cowan`. That alone is enough to clone. Note the resulting absolute path
   (e.g. `C:\dev\tournament-website` → hash `c--dev-tournament-website`); you'll
   match the `.claude\projects` folder to it in Stage 1.5.
5. **Claude Code** — open the cloned folder in VS Code, install the **Claude
   Code** extension from the marketplace (or `npm i -g @anthropic-ai/claude-code`).
6. **Log in** — launch Claude, run `/login`, complete in the browser. (Cleaner
   than copying `.credentials.json`.)

✅ Claude is now running in the repo and can take over.

---

## Stage 1.5 — Restore history & memory

1. Copy the saved `c--Users-Robert-Cowan-Documents-tournament-website` folder
   into `C:\Users\<you>\.claude\projects\`.
2. Same username + same repo path → done, no rename.
3. Different username → rename the folder to match the new path using the
   encoding rule above (lowercase drive; `:`, `\`, space → `-`). Claude can do
   this rename for you.
4. Restore `settings.json`, `commands\`, and `plans\` into `~\.claude\`.

---

## Stage 2 — Claude-assisted installs

Hand these to Claude in the VS Code terminal:

- **pnpm** (active package manager — `node_modules/.pnpm` + `pnpm-lock.yaml`):
  `npm i -g pnpm`
- **AWS CLI v2:** `winget install Amazon.AWSCLI`
- **GitHub CLI:** `winget install GitHub.cli`
- **Project deps:** `pnpm install` (from repo root)
- **Playwright browsers** (only if running `/uat`): `npx playwright install`

---

## Stage 3 — Secrets & auth (you provide values)

1. **Env files** — drop the copied `.env.local` and `.env.production.local`
   into the repo root. Do not rebuild by hand. `.env.local` keys for reference:
   `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
   `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_ACCESS_TOKEN`, `RESEND_API_KEY`,
   `NEXT_PUBLIC_APP_URL`, `STRIPE_SECRET_KEY`,
   `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET`,
   `PLATFORM_ADMIN_EMAILS`, `UNSUBSCRIBE_SECRET`, `NEXT_PUBLIC_VAPID_PUBLIC_KEY`,
   `VAPID_PRIVATE_KEY`, `VAPID_EMAIL`, the `NEXT_PUBLIC_*` feature toggles, and
   the `UAT_*` set.
2. **AWS** — run `aws configure` fresh on the new machine (creates the standard
   `.aws\credentials` file — preferred over re-creating env vars). Region
   `us-east-2`, output `json` (Amplify app `d3ld0l2bgmmlga`; `/release` passes
   the region explicitly anyway). If you can't retrieve the existing secret key,
   **rotate it**: create a new access key in the IAM console, configure with it,
   then deactivate the old one. Verify: `aws sts get-caller-identity`.
3. **GitHub CLI auth** (for PR/gh ops): `gh auth login` → HTTPS → browser.
4. **Git identity** (if not carried over):
   ```powershell
   git config --global user.name "b2cowan"
   git config --global user.email "b2cowan@gmail.com"
   ```

---

## Stage 4 — Verify

1. **Dev server** (needs real network for Supabase): `pnpm dev`
2. **Confirm** `http://localhost:3000/platform-admin/login?next=%2Fplatform-admin`
   returns **HTTP 200** with no Supabase `EACCES` errors in the server log.
3. **Static checks:** `pnpm typecheck` (clean = toolchain good).
4. **Memory sanity check:** ask Claude something only the restored memory knows;
   a correct answer confirms history/memory mapped.

---

## Notes

- **VS Code Settings Sync** (sign in with GitHub/Microsoft) pulls your other
  extensions — separate from Claude.
- **No local Postgres** — Supabase is cloud; scripts use `supabase-js` with the
  service-role key from the env files.
- **Stripe CLI** is not a project dependency; install only if testing webhooks
  locally.
- `RELEASE_CONFIG.md` ships with the clone, so `/release` works once AWS creds
  are in place.
