# Standalone Team Workspace

- 2026-05-25 update: this implementation is now the premium entitlement layer for the unified customer-facing **Coaches Portal** product. Public copy should say Coaches Portal rather than Team. Basic tournament-only coach records and premium Team workspace tools now belong in one portal; see `memory/coaches-portal-unified-product.md` and `docs/projects/active/COACHES_PORTAL_UNIFIED_PROJECT_PLAN.md`.
- Migration `065_team_workspace_foundation.sql` has been applied in dev and production.
- Phase 2A is complete: `lib/team-workspace-provisioning.ts` provisions the lightweight Team workspace org, rep team, active program year, coach assignment, workspace row, entitlement row, and ledgers.
- Phase 2B is complete for plumbing: direct Team checkout, mock provisioning, Stripe metadata, webhook recovery, subscription sync, and checkout completion redirect shell are implemented.
- Phase 2C is complete: Team workspaces default to `/{orgSlug}/coaches`, the coaches landing/success state uses Team-first copy, coach assignments for Team workspaces are filtered through active `team_entitlements`, and org-wide rep-team admin remains protected by `module_rep_teams`.
- Phase 2D is complete: the legacy public Team signup surface creates coach auth users without creating a normal org, and the page starts the existing Team checkout flow after account creation/sign-in. Pricing, marketing navigation, and the footer linked to this path before Coaches Portal unification.
- Phase 2E is complete: `/api/dev/team-checkout-readiness` and Platform Admin Dev Tools check Team checkout readiness, including app URL, webhook secret, Team gating, Stripe environment, and Team monthly/annual price metadata when a Stripe key is configured. Dev Tools also has a dev-only runtime Mock Billing toggle that can enable/disable mock billing without editing `.env.local`; it resets on dev server restart. Real Stripe sandbox org Team add-on checkout/webhook smoke passed on 2026-05-23 via `tests/uat/scenarios/team-org-billing-stripe-smoke.spec.ts`.
- Phase 3A is complete: secure tournament Team claim links can be generated for tournament teams, public registration confirmation emails include a claim CTA for non-waitlisted teams, the legacy claim route prefills the Team activation screen, checkout verifies the signed-in email matches the claim contact, provisioning records the source tournament/team, and claims are marked claimed only after workspace creation.
- Phase 3B is complete: tournament organizers can select teams from the tournament teams admin screen and send Team workspace claim invitations. The admin API creates fresh secure links for pending/accepted teams, emails eligible contacts, skips missing-email/waitlist/rejected/already-claimed teams, and returns generated links for organizer follow-up.
- Phase 4A is complete: standalone Team coaches can request a Basic visibility link to a parent organization from the Coaches Portal, and org owners/admins can approve or decline from Organization Admin. Approval marks the link/workspace as linked but does not transfer billing, ownership, roster/doc/accounting access, or org-wide rep-team admin access.
- Phase 4B is complete: organization owners/admins can invite a standalone Team workspace to a Basic visibility link by workspace slug or primary coach email, and the Team coach can accept or decline from the Coaches Portal. Acceptance marks the link/workspace as linked using the Phase 4A state update and still does not transfer billing, ownership, roster/doc/accounting access, parent-org `team_entitlements`, or org-wide rep-team admin access.
- Phase 4C is complete: after a Basic link is active, Team coaches can request org billing or accept an org billing invitation, and org owners/admins can approve and complete org Team add-on checkout/mock application. Completion moves `team_workspaces.billing_mode` to `org_team_addon`, sets `billing_owner_org_id`, keeps a workspace-org active Team entitlement for coach access, records a parent-org team entitlement for the add-on, and preserves Basic sharing without transferring ownership, roster/doc/accounting access, or org-wide rep-team admin access. Mock and real Stripe sandbox smoke paths are passing.
- Phase 5A is complete for the approval foundation: after a link is active, Team coaches can request ownership transfer, org owners/admins can invite or approve ownership transfer, and coaches can accept or decline org invitations. Mutual approval sets `team_org_links.status = 'ownership_pending'`, `link_type = 'ownership'`, and `sharing_level = 'full_org_owned'` as an auditable platform-assisted state; final `org_id` reassignment/ledger transfer remains the next Phase 5 slice.
- Phase 5B is complete and smoke-tested: `supabase/migrations/067_team_ownership_transfer_rpc.sql` has been applied in dev and production, and the focused UAT smoke `tests/uat/scenarios/team-ownership-transfer-smoke.spec.ts` passed on May 23, 2026. The RPC adds `complete_team_workspace_ownership_transfer`, and platform admin org detail can complete mutually approved transfers. Completion moves team-scoped rep-team rows and team ledger ownership to the linked org, creates/activates coach memberships there, retires Team entitlements, marks the link/workspace org-owned, suspends retired workspace-org memberships, and attempts Stripe cancellation from server code when a real prior subscription exists.
- Ownership transfer rollback decision for MVP: no dedicated rollback RPC. Transfers are expected to be rare and platform-assisted; failed pre-completion attempts are retryable, while completed reversals are support/engineering assisted case by case.
- Phase 6A is complete and smoke-tested for coach-managed attendance: migration `069_rep_team_event_attendance.sql` has been applied in dev and production, and `tests/uat/scenarios/team-attendance-smoke.spec.ts` passed on May 23, 2026. It adds event-to-player attendance rows and a scope-sync trigger so attendance follows event rows during ownership transfer. Coaches Portal schedule event details let coaches mark active roster players In, Out, Late, or Unknown with optional notes. This remains coach-scoped and does not expand linked-org Basic visibility or ownership boundaries. The smoke found and fixed a Next 16 client params issue in the coaches schedule page.
- Phase 6B is complete and smoke-tested: migration `070_rep_team_lineups.sql` and the follow-up `071_rep_team_lineups_rls_fix.sql` have been applied in dev and production, and `tests/uat/scenarios/team-lineup-smoke.spec.ts` passed on May 23, 2026. It adds roster position fields plus event lineup tables, RLS read policies, and event scope sync. Coaches can store primary/secondary positions, export roster positions, and build lineup cards from game/scrimmage event details in either `Everyone bats` or `9 player ball` mode. Everyone bats keeps every active player in the batting order and allows `Bench` as an inning position; 9 player ball separates nine starters from bench players while still allowing inning-by-inning substitution planning.
- Phase 6C is implemented and smoke-tested: the coach team overview now uses the Next 16 client params pattern and shows a data-driven `Season setup` checklist for roster, jersey/position setup, calendar events, lineup readiness, budget setup, and optional parent-org linking for standalone Team workspaces. It reuses existing coach-scoped roster/events/budget APIs and does not expand linked-org visibility, billing, ownership, roster/document/accounting access, or org-wide rep-team admin rights.
- Phase 7A is implemented and smoke-tested: public `/pricing` now starts with segment choices for tournament organizers, league/club leaders, and coaches managing one competitive team. The legacy Team path presents $290 CAD seasonal pricing first with $29 CAD month-to-month as secondary, and highlights roster, schedule, dues, documents, budget, attendance, lineups, setup checklist, reminders, optional parent-org linking, and one free-tier local tournament slot. Organization plan cards remain in the shared org pricing component, so normal org onboarding is unchanged.
- Phase 7B is implemented and smoke-tested: the legacy public Team route reads as a coach-specific landing page around the existing signup form, with "From tournament weekend to season workspace" positioning, tournament-team continuity, Coaches Portal value, attendance/lineup/budget/document messaging, parent-org readiness, and quick local tournament creation. Signup and checkout plumbing are unchanged.
- Phase 7C is implemented and smoke-tested: tournament-to-Team CTA copy now appears in registration confirmation, public tournament acquisition banners, registration/claim emails, and post-event results emails. Secure claim links remain email-only for registration submissions, while public CTAs carried source/org/tournament context. Existing organizer tournament CTAs are preserved.
- Phase 7D is implemented: Team Subscription points coaches to the existing Basic parent-org link and org billing approval flow, while Organization Subscription and Team Links show a Club value nudge when an org has 3+ active org-paid Team add-ons. These are informational upsells only and do not change billing, ownership, linked-org data visibility, roster/document/accounting access, or org-wide rep-team admin rights.
