# Help Documentation Planning Rule

- Whenever a project, feature, module, or significant workflow change is contemplated and an implementation plan is created, the plan must explicitly consider whether customer-facing and/or platform-admin help documentation needs to be created or updated.
- Help documentation should be treated as part of the feature scope, not as an optional cleanup item after launch.
- For customer-facing changes, consider task-based help articles, troubleshooting notes, role/access explanations, launch checklists, and related-guide links.
- For platform-admin changes, consider employee-only SOPs, support workflows, audit/billing/product governance notes, and customer-support talking points.
- If no help updates are needed, the implementation plan should say why.
