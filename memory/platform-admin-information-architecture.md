# Platform Admin Information Architecture

- Phase 1.5 exists because the organization detail page became a long stack of unrelated support, billing, entitlement, people, and activity panels.
- The org detail layout now prioritizes a compact account snapshot, a needs-attention strip, primary actions, and grouped workflow tabs.
- The org detail readability pass now stacks snapshot label/value pairs, allows long emails and ids to wrap, and gives workflow tables more spacing so member and tournament rows do not look compressed.
- On May 20, 2026, the platform-admin-wide Phase 1.5 audit was documented across Overview, Organizations, Customer Users, Retention Queue, Plans & Pricing, Early Access, Platform Users, and Audit Log.
- The Organizations directory now has an account snapshot, a needs-attention strip, a primary account-finding panel, a Customer Users shortcut, and a grouped account directory table.
- The Organizations directory is intentionally read-only: plan overrides and tournament-limit changes should happen from org detail workflows, not directly from directory rows.
- Phase 1 and Phase 1.5 are now considered complete. Structured timestamped org notes are added to Platform Admin Phase 3 scope so support notes become a durable add/edit/delete history instead of one mutable `organizations.internal_notes` field.
- Phase 2 initial dashboard slice is implemented on Overview with live-query totals, subscription health, growth, early-access status, product usage, estimated MRR/ARR, and action alerts. Historical churn/downgrade/recovery/trend accuracy still needs durable events and snapshots.
- Phase 2 expanded live-query dashboard slice adds new organizations by plan, early-access conversion rate, top source paths, owner inactivity, and tournaments-created-in-30-days. Migration 052 adds `tournaments.created_at` and has been applied in dev and production.
- Phase 2 durable lifecycle slice adds `platform_events` via migration 053 plus event writes from billing cancel/downgrade routes, Stripe webhooks, and mock billing. Overview now reports cancellations, downgrades, recoveries, and 30-day past-due recovery rate from events. Migration 053 has been applied in dev and production.
- The Overview dashboard keeps top totals and the action queue visible, then groups deeper metric panels into Subscription, Growth, Usage, and Metric Notes tabs to reduce long-scroll clutter.
- Phase 2 is complete: migration 054 for `platform_metric_snapshots` and `platform_admin_visits` has been applied in dev and production, Overview reads the previous platform-admin visit to show "past due since visit", and the snapshot/visit flows have been browser-verified.
- Phase 3 initial Plans & Pricing safety slice is implemented and browser-verified: subscriber impact appears before availability, limits/trial, and Stripe price edits; last-change notes are stored for those config surfaces; plan availability, plan config, and Stripe price ID updates now write platform audit entries with old/new payloads.
- Phase 3 structured org notes are implemented and browser-verified behind migration 054: `org_internal_notes` stores add/edit/soft-delete note history, existing `organizations.internal_notes` values are backfilled as initial notes, and org detail Support now manages individual timestamped notes with audit entries and app-native delete confirmation.
- Phase 3 billing/product safety is complete and browser-verified: it includes Stripe price validation when the configured Stripe secret key matches the target environment, org plan/limit changes in the Billing & Access workflow with reason plus app-native confirmation, override writes that no longer mutate the base subscription status, and Audit Log investigation tools for CSV export, org filtering, action labels, and full JSON value viewing.
- Phase 4 initial governance slice is complete and browser-verified: platform roles (`super_admin`, `support`, `billing`, `product`, `read_only`) and permission gates cover platform-user management, support actions, billing/retention/override actions, product/add-on/config actions, and role-scoped org-detail/platform-user controls. Migration 055 has been applied in dev and production.
- Platform admin confirmation flows should use in-app modals or inline confirmation states, not native browser `confirm`/`alert`/`prompt` dialogs.
- Migration 051 (`plan_pricing_change_notes`) has been applied in dev and production.
- Next platform-admin phase is Phase 5: Growth and Product Catalog, starting with early-access conversion tracking.
- Phase 5 Early Access conversion tracking slice is implemented and browser-verified behind migration 057: leads can be marked converted with `converted_at`, linked to a converted organization, assigned a follow-up due date, and given a next-action note. The Early Access page now shows conversion breakdowns by plan and feature interest, and CSV export includes conversion/follow-up fields. Migration 057 has been applied in dev and production.
- Phase 5 Product Catalog read-only foundation is implemented and browser-verified behind migration 058: Plans & Pricing has a Product Catalog tab for plan versions, add-on catalog records, and a feature matrix derived from `PLAN_CONFIG`. Migration 058 has been applied in dev and production.
- Phase 5 Product Catalog governance slice is implemented and browser-verified behind migration 059: Product Catalog now tracks planned catalog change requests, approval/status workflow records, and coupon/promo/trial/launch/retention campaign records without mutating live pricing or entitlements. Migration 059 has been applied in dev and production.
- Phase 5 approval enforcement is implemented and browser-verified behind migration 060: live plan availability, limits/trials, and Stripe price ID changes now require an approved Product Catalog change request selected in Plans & Pricing, and each live application is logged in `platform_catalog_change_applications`. Migration 060 has been applied in dev and production.
- Phase 5 Feature Matrix draft editor is implemented and browser-verified: product-capable admins can toggle proposed plan/module entitlements in Plans & Pricing and save them as `feature_matrix` change requests with the proposal stored in `platform_catalog_change_requests.proposal`. This is proposal-only and does not mutate live entitlements.
- Phase 5 Feature Matrix publishing is implemented and browser-verified behind migration 062: approved `feature_matrix` change requests can be previewed, published with a note into `platform_plan_module_entitlements`, marked implemented, and logged in both `platform_audit_log` and `platform_catalog_change_applications` with surface `feature_matrix`. Migration 062 has been applied in dev and production.
- Phase 5 Bulk Operations foundation is implemented and browser-verified behind migration 063: `/platform-admin/bulk-operations` lets billing-capable platform admins filter/select organizations, preview guarded actions, enter a required reason, confirm in-app, and run bulk subscription status overrides, comp-period grants, and base plan changes. Each run writes `platform_bulk_operations` plus per-org `platform_audit_log` entries. Migration 063 has been applied in dev and production.
- Phase 5 Bulk module/add-on enablement is implemented and browser-verified behind migration 064: product-capable platform admins can bulk enable or remove organization-specific module add-ons from `/platform-admin/bulk-operations`, with current-state preview, required reason, inline confirmation, batch logging, and per-org audit entries. Migration 064 has been applied in dev and production.
- Grandfathering rules are deferred because FieldLogicHQ does not yet have active/paying users that need legacy pricing or entitlement treatment.
- Phase 6 readability QA has started with a shared shell cleanup that is browser-verified: the Platform Admin sidebar is sticky/scrollable on desktop, stacks into a top navigation area on narrower screens, supports horizontal nav scrolling, and reduces main content padding for mobile scanability.
- Phase 6 header/action-label cleanup is implemented: Retention Queue, Platform Users, and Audit Log now use job-area labels instead of generic product labels, and Audit Log has human-readable labels for bulk operation audit actions.
- Phase 6 Plans & Pricing density cleanup is implemented and browser-verified: the Product Catalog tab now has inline Planning, Feature Matrix, and Catalog Records workspaces so change requests/campaigns, entitlement matrix work, and baseline catalog records do not stack into one long scroll.
- Per user direction on May 21, 2026, browser verification should not be carried as a separate task item for the remaining Platform Admin cleanup unless explicitly requested.
- Platform Admin Phases 1 through 6 are complete for launch scope. The remaining Phase 6 watch items are deferred to future usage-driven cleanup: table-heavy Customer Users/Audit Log, Org Detail duplicate facts after real support use, future Bulk Operations history drill-in, Early Access growth-field separation, and Retention Queue hard-purge policy copy.
- The Platform Admin implementation plan and PM brief have been archived to `docs/archive/merged_PLATFORM_ADMIN_IMPLEMENTATION_PLAN.md` and `docs/archive/PLATFORM_ADMIN_PM_BRIEF.md`.
